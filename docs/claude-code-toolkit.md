# Инструменты разработки (Claude Code) для ReelScribe

Это твоя учебная карта: какие современные возможности Claude Code применяем в проекте, **зачем именно
здесь**, **как включить** и **как это работает**. Claude Code обязан объяснить каждый инструмент перед
первым применением (правило «Режим обучения» в `CLAUDE.md`) — этот файл его шпаргалка и твоя памятка.

> Источник правды о новых возможностях — официальный changelog: https://code.claude.com/docs/en/changelog
> Если что-то из описанного у тебя в версии называется иначе — проверь там и в `/help`. Версию смотри `claude --version`.

| Этап ReelScribe | Инструмент | Зачем именно здесь |

## 1. Субагенты (параллельная работа)
**Что.** Отдельные под-агенты, которым основной агент поручает куски работы; они работают параллельно и
могут сами звать своих под-агентов (в глубину). У каждого свой контекст.
**Зачем здесь.** В ReelScribe три слабо связанные части: бэкенд-пайплайн, фронт-таблица, слой ASR/downloader.
Их удобно вести параллельно — пока один пишет воркер, другой собирает таблицу.
**Как включить.** Попросить словами: «сделай это двумя субагентами — один на backend-пайплайн, другой на
frontend-таблицу». Панель агентов видна в интерфейсе; `claude agents` — общий список сессий.
**Как работает.** Каждый субагент — отдельный прогон со своим окном контекста; результаты возвращаются
основному. Изоляция контекста = меньше путаницы, но больше расход токенов и памяти.
**Осторожно (M1, 8 ГБ):** держим 2–3 максимум. faster-whisper, ffmpeg и сборка фронта сами тяжёлые —
не запускать их в нескольких агентах одновременно.

## 2. `/effort` — уровень усилия модели
**Что.** Регулятор «быстрее ↔ умнее». Чем выше, тем глубже модель думает (и дольше/дороже).
**Зачем здесь.** Проектирование интерфейсов стратегий (ASR/downloader), модели данных и пайплайна —
дать `high`/`xhigh`. Бойлерплейт и мелкие правки — обычный уровень, чтобы не жечь время.
**Как включить.** Команда `/effort` (выбрать уровень), либо `/effort xhigh` для конкретного тяжёлого шага.
**Как работает.** Поднимает бюджет «рассуждения» перед ответом. На простом — переплата, на сложном — окупается.

## 3. Plan mode — план перед действиями
**Что.** Режим, где агент сначала выдаёт план, ничего не меняя в файлах.
**Зачем здесь.** Перед крупной фичей (весь пайплайн, экран результатов) сверить план с ТЗ и прототипом
до того, как польётся код.
**Как включить.** Переключатель режима (Shift+Tab циклом) или попросить «сначала план, без правок».
**Как работает.** Агент ограничен в инструментах правки, пока ты не подтвердишь план.

## 4. Supabase через MCP — прямой доступ к БД
**Что.** MCP-коннектор даёт агенту инструменты к внешнему сервису. У тебя уже подключён Supabase.
**Зачем здесь.** Вся персистентность ReelScribe в Supabase (§5 ТЗ): таблицы `reels`, `transcripts`,
`import_sessions`, `jobs`. Агент может создавать схему, гонять миграции и проверочные запросы напрямую.
**Как включить.** Коннектор уже есть; проверить — `/mcp` (список серверов и их инструментов).
**Как работает.** Агент вызывает инструменты MCP-сервера (запрос/DDL) — это реальные действия в твоей базе.
**Осторожно.** Перед DDL и миграциями агент обязан показать, что выполнит, и дождаться твоего «да».
RLS включить (данные приватны), ключи — только в env.

## 5. Хуки — автоматический контроль качества
**Что.** Скрипты, которые Claude Code запускает на события (например, после каждой правки файла).
**Зачем здесь.** Чтобы линтер и быстрые тесты бэкенда гонялись сами — не надо помнить про ручной прогон.
**Как включить.** Файл `.claude/settings.json`, секция `hooks` (готовый пример уже в репозитории).
Проверить корректность — `/doctor`.
**Как работает.** На совпадение по событию/инструменту хук запускает команду; её результат возвращается
агенту (и он чинит, если что-то упало). У нас: `ruff` после правок бэкенда + быстрые `pytest` в конце хода.

## 6. `/code-review` — ревью с авто-фиксами
**Что.** Встроенное ревью кода с выбором глубины; может сразу применить найденные фиксы.
**Зачем здесь.** Перед коммитом ключевых кусков (пайплайн, троттлинг, идемпотентность) и перед деплоем
на Render — отловить баги и упрощения.
**Как включить.** `/code-review` (на сложных местах `/code-review high`).
**Как работает.** Отдельный проход модели-ревьюера по диффу/коду; находки можно применить в рабочее дерево.

## 7. `/goal` — автономная работа до цели
**Что.** Задаёшь условие «готово», и агент продолжает работать через несколько ходов, пока не выполнит.
**Зачем здесь.** Финальная добивка MVP: цель = «выполнены критерии приёмки §15 ТЗ и `pytest` зелёный».
Не пинать вручную каждый шаг.
**Как включить.** `/goal` и сформулировать измеримое условие.
**Как работает.** Держит цель между ходами, показывает прогресс; останавливается по достижении или по тебе.
**Не путать:** `/goal` — про *сборку приложения*. Прогон 873 рилсов — это работа самого приложения
(воркер очереди), а не задача Claude Code.

## 8. Динамические воркфлоу (`/workflows`)
**Что.** Просишь собрать воркфлоу — агент оркестрирует работу через много под-агентов в фоне.
**Зачем здесь.** Только для реально крупного: большой рефактор структуры, разнос `CLAUDE.md` по
`.claude/rules/*.md` (как предлагает §16 ТЗ). Для обычной фичи — избыточно.
**Как включить.** `/workflows`; запуск — явной фразой (триггер-слово подсвечивается в поле ввода).
**Как работает.** Дерево агентов выполняет шаги параллельно/последовательно; прогон виден в `/workflows`.

## 9. Auto mode — безопасное само-одобрение
**Что.** Классификатор сам пропускает безопасные действия и блокирует опасные, не дёргая тебя на каждое.
**Зачем здесь.** Чтобы рутинные правки/чтения шли без кликов, но удаление, деплой и миграции — со стопом.
**Как включить.** Переключатель режима (Shift+Tab циклом). Опасные git-команды защищены отдельно.
**Как работает.** Перед действием встроенный классификатор оценивает риск; необратимое требует подтверждения.

## 10. Скиллы — твои конвенции, которые агент подхватывает сам
**Что.** Папки с `SKILL.md`: инструкции/процедуры, которые Claude Code применяет автоматически по контексту.
**Зачем здесь.** Зафиксировать правила ReelScribe (видео не хранить, кэш по shortcode, UI строго по
прототипу, метрики по формулам §9), чтобы не повторять их в каждом промпте.
**Как включить.** Лежит в `.claude/skills/<name>/SKILL.md` (пример — `reelscribe-conventions`). Скаффолд
нового — `claude plugin init <name>`. Перечитать без рестарта — `/reload-skills`.
**Как работает.** Агент видит описания скиллов и подтягивает нужный, когда задача совпадает с триггером.

## Порядок освоения (рекомендация)
1. **Хуки + Supabase MCP** — базовая гигиена и доступ к БД. Завести на старте.
2. **plan mode + `/effort`** — на проектировании модели данных и интерфейсов.
3. **Субагенты** — когда пойдут параллельно бэкенд и фронт.
4. **`/code-review`** — перед первым коммитом крупного куска.
5. **`/goal`** — на финишной добивке MVP.
6. **Скиллы** — закрепить конвенции, когда устанешь повторять их руками.
7. **Динамические воркфлоу** — позже, на первом большом рефакторе.

Не обязательно всё сразу: бери по 1–2 на этап, и проси объяснить перед первым применением.
