# Конвенции ReelScribe

Полное ТЗ — `docs/T3_ReelScribe.md`. Визуальный эталон — `reelscribe_v12.html`. Этот скилл — выжимка
правил, которые нельзя нарушать.

## Железные правила
- **Видео не хранить.** Качать только аудиодорожку; после распознавания аудиофайл удалять. В БД — текст и метрики.
- **Дедуп и кэш по `shortcode`.** Есть успешная расшифровка у shortcode → не качать и не распознавать заново.
- **ASR работает из РФ без иностранной оплаты/VPN.** Дефолт — faster-whisper локально (`medium` на M1).
  Облачный фолбэк — Yandex SpeechKit. `large-v3` — батч на внешнем GPU, не как обязательный платный сервис.
- **Скачивание троттлим.** 1–2 параллельно, паузы с джиттером, backoff-ретраи, фолбэк на Apify.
- **Изоляция ошибок по рилсу.** Сбой одного (приватный/удалён/нет аудио) не валит партию: `failed` + причина.
- **Ранний показ.** Первая партия (~50) открывается сразу, остальное — в фоне.

## Данные и термины (не путать)
- `caption` = текст поста (подпись автора); `transcript` = расшифровка речи. Это разные поля.
- Заметки двух уровней: `reel_notes.note` (по рилсу) и `import_sessions.comment` (по партии).
- Метрики по формулам §9 ТЗ: `er = views/followers` (🔥 ≥10 оранж, 5–10 зелёный, <5 серый),
  `lpf = likes/followers·100%`, `cpf = comments/followers·100%`, `eng = (likes+comments)/views·100%`.
  Считать на чтение, не рассинхронизировать.

## Код
- Backend: type hints везде; pydantic-схемы для API; бизнес-логика в `services/`, доступ к данным — не в роутерах;
  стадии пайплайна идемпотентны и логируют прогресс; async/await на I/O.
- Frontend: функциональные компоненты + хуки; **без `localStorage`/`sessionStorage` для прод-хранения** —
  состояние в React-стейте на сессию, постоянное — в Supabase; запросы к API в `src/api/`.
- UI и заголовки экспортных колонок — на русском (в JSON-экспорте — техн. англоключи).
- Меняешь UI — сверяйся с `reelscribe_v12.html`, не выдумывай свой вид.

## Безопасность
- Секреты только в env (`.env` в `.gitignore`), никогда в коде, в этом файле или в БД.
- Supabase RLS включён. Перед DDL/миграциями — показать план и дождаться подтверждения.
