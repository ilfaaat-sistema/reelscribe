# ReelScribe

Сервис: берёт ссылки на Instagram Reels/посты → скачивает аудио → расшифровывает речь →
переводит иностранные на русский → подтягивает метрики (просмотры/лайки/автор/подписчики) →
показывает в таблице с аналитикой и отдаёт на экспорт.

Этот файл — конституция проекта. Подробное ТЗ: `docs/T3_ReelScribe.md`.
Визуальный эталон UI/UX (макет, поведение, расположение): `reelscribe_v12.html` — открой в браузере и **повторяй его**.

Как строить этот проект современными инструментами Claude Code (субагенты, хуки, /goal, /code-review,
Supabase MCP) и как они работают — `docs/claude-code-toolkit.md`. **Перед первым применением любого
из этих инструментов сначала объясни мне его** (см. «Режим обучения» ниже).

## Стек
- Backend: **Python 3.11 + FastAPI**, фоновый воркер на очереди.
- БД: **Supabase (Postgres)**.
- Скачивание: **yt-dlp + ffmpeg** (только аудио). Фолбэк метаданных: Apify.
- ASR: **faster-whisper** (локально). Облачные движки опционально: Yandex SpeechKit, Deepgram, OpenAI.
- Frontend: **React + Vite**, без UI-кит-зависимостей сверх необходимого.
- Деплой: **Render**. Секреты — только в env, никогда в коде и не в этом файле.

## Железные правила
- **YOU MUST: видео не хранить.** Качаем только аудиодорожку, после распознавания аудио удаляем. В БД — только текст и метрики.
- **YOU MUST: дедуп и кэш по `shortcode`.** Если у shortcode уже есть успешная расшифровка — НЕ скачивать и НЕ распознавать заново, брать из БД.
- **IMPORTANT: ASR должен работать из РФ без иностранной оплаты/VPN как жёсткой зависимости.** По умолчанию — faster-whisper локально. Облачный фолбэк — Yandex SpeechKit. Модель `large-v3` гоняется батчем на бесплатном GPU (Kaggle ~30 ч/нед), а не как обязательный платный сервис.
- **IMPORTANT: скачивание троттлим.** 1–2 параллельно, паузы с джиттером между запросами, чтобы Instagram не блокировал. При блоке/ошибке — ретрай с backoff, затем фолбэк на Apify.
- Весь UI и заголовки экспортных колонок — **на русском** (в JSON-экспорте оставляем технические англоключи).
- Результаты показывать **рано**: после первой партии (~50) открывать таблицу, остальное догружать в фоне.
- Обработка по-рилсово изолирована: ошибка одного (приватный/удалён/нет аудио) не валит партию — помечаем `failed` + причина.

## Терминология и метрики (считать ровно так)
- `shortcode` — id рилса из URL `instagram.com/(reel|p|tv)/<shortcode>`.
- **Залётность** `er = просмотры ÷ подписчики`. Цвет: оранжевый ≥10×, зелёный 5–10×, серый <5×.
- **Лайки/подп** `lpf = лайки ÷ подписчики` (в %).
- **Комм/подп** `cpf = комментарии ÷ подписчики` (в %).
- **Вовлечённость** `eng = (лайки + комменты) ÷ просмотры` (в %) — доля посмотревших, кто отреагировал.
- **Текст поста** = `caption` (подпись автора). **Расшифровка** = `transcript` (распознанная речь). Это РАЗНЫЕ поля, не путать.
- Два уровня заметок: `note` у рилса (про одно видео) и `comment` у сессии импорта (про всю партию).

## Конвенции
- Backend: type hints везде, pydantic-схемы для API, бизнес-логика в `services/`, в роутерах — только валидация/вызов. SQL/доступ к данным — не в роутерах.
- Frontend: функциональные компоненты + хуки, состояние локально (без localStorage — см. ТЗ), запросы к API в `src/api/`.
- Async/await на I/O. Идемпотентность импорта по shortcode.
- Меняешь поведение UI — сверяйся с `reelscribe_v12.html`, не выдумывай свой вид.

## Как строить с современным Claude Code
Эти правила — про то, *как* вести разработку, чтобы задействовать свежие возможности. Детали и
объяснения каждого — в `docs/claude-code-toolkit.md`.

- **Параллелизм через субагентов — но скромно (M1, 8 ГБ): максимум 2–3 одновременно.** Делю независимые
  части между субагентами: бэкенд-пайплайн ↔ фронтенд-таблица ↔ ASR/downloader-стратегии. Не запускаю
  веер агентов на тяжёлых задачах (faster-whisper/ffmpeg сами едят память).
- **Усилие под задачу.** Архитектурные и «думательные» шаги (интерфейсы стратегий ASR/downloader,
  модель данных, пайплайн) — `/effort high`, самые сложные — `/effort xhigh`. Рутина (бойлерплейт,
  правки по мелочи) — обычный уровень.
- **План перед крупным.** Перед большой фичей — сначала plan mode (план в ответ, без правок), сверяемся,
  потом исполнение.
- **БД — через Supabase MCP.** Схему, миграции и проверочные запросы делаю через подключённый Supabase
  MCP, а не руками вслепую. Перед DDL/миграцией — показать, что собираюсь выполнить, и дождаться «да».
- **Качество в конвейере.** Линтер/тесты гоняются хуками автоматически (см. `.claude/settings.json`).
  Перед коммитом и перед деплоем на Render — `/code-review` (на сложных местах `high`), фиксы применить.
- **Автономная добивка.** Когда цель чётко измерима (например «все критерии приёмки MVP из §15 ТЗ
  выполнены и тесты зелёные») — использую `/goal`, чтобы доработать через несколько ходов без ручного
  пинка на каждом шаге.
- **Крупные рефакторы — динамические воркфлоу** (`/workflows`), но только когда задача реально большая
  и разбивается на десятки шагов; для обычных фич это избыточно.
