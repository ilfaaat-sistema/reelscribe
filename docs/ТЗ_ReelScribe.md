# Техническое задание — ReelScribe

> Документ для реализации через Claude Code. Визуальный эталон поведения и вёрстки —
> кликабельный прототип `reelscribe_v12.html` в корне проекта. Везде, где описание расходится
> с прототипом, **прототип главнее по части UI/UX**; это ТЗ главнее по части данных, API и логики.

---

## 1. Что это и зачем

ReelScribe превращает сохранённые Instagram Reels (и посты) в базу расшифровок с аналитикой.
Пользователь скидывает ссылки (или экспорт переписки Instagram `message_1.json`), сервис скачивает
аудио, распознаёт речь, переводит иностранное на русский, подтягивает метрики (просмотры, лайки,
комментарии, автор, число подписчиков) и показывает всё в таблице с режимом аналитики, карточкой
каждого рилса и экспортом. Цель — быстро находить «залетевшие» идеи и пересматривать/переснимать их.

**Пользователь:** контент-мейкер/аналитик, работает на русском, на MacBook M1 (8 ГБ), из РФ.
Поэтому критичны: работа ASR без иностранной оплаты/VPN, экономный по памяти прогон, русский интерфейс.

---

## 2. Пользовательские сценарии

1. **Массовый разбор архива.** Кидает `message_1.json` (экспорт «Сохранённого»/переписки) → сервис
   парсит ~870 ссылок, дедуплицирует, ставит в очередь, показывает первую партию результатов уже
   через секунды, остальное догружает в фоне.
2. **Точечный импорт.** Вставляет 10–40 ссылок или `.txt`/`.csv` → быстрый прогон.
3. **Анализ.** Сортирует по залётности/просмотрам, фильтрует по автору и порогам, видит «залётные».
4. **Глубокое погружение.** Кликает рилс → карточка с превью, текстом поста, расшифровкой (+перевод),
   метриками, саммари; оставляет личную заметку «переснять так-то».
5. **Чтение подряд.** Режим «Чтение» — листает расшифровки одну за другой.
6. **Выгрузка.** Экспорт в CSV/Excel/JSON/Markdown (русские заголовки, текст на русском по умолчанию)
   или заливка в Supabase/Notion.

## 3. Архитектура

```
Импорт ссылок ─▶ normalize ─▶ dedup(shortcode, кэш) ─▶ очередь
                                                          │
                              ┌───────────── воркер ──────┘
                              ▼
 download(yt-dlp, троттл) ─▶ extract_audio(ffmpeg) ─▶ transcribe(faster-whisper)
   │ при блоке → Apify фолбэк                          │
   ▼                                                   ▼
 metadata(просмотры/лайки/автор/подписчики)   translate(если не RU) ─▶ summarize(опц.) ─▶ OCR(опц.)
                              │                                   │
                              └──────────── persist (Supabase) ───┘ ─▶ cleanup(удалить аудио)
```

Фронтенд опрашивает прогресс (polling или SSE) и рендерит результаты по мере готовности.

## 4. Стек и инфраструктура

- **Backend:** Python 3.11, FastAPI, Uvicorn. Фоновый воркер — отдельный процесс, читает очередь.
- **БД:** Supabase (Postgres). Доступ через supabase-py или SQLAlchemy.
- **Скачивание/аудио:** yt-dlp + ffmpeg (извлекать только аудио, формат `.m4a`/`.wav` mono 16 kHz).
- **ASR:** faster-whisper (CTranslate2). Модели `medium` (дефолт, идёт на M1) и `large-v3` (батч на GPU).
- **Метаданные:** парсинг из ответа yt-dlp где возможно; фолбэк — Apify Instagram scraper (грей-зона ToS, опционально).
- **Перевод/саммари:** дешёвый LLM (через API). Перевод EN→RU и краткое саммари.
- **Frontend:** React + Vite. Без тяжёлых UI-китов; стили — как в прототипе (тёмная тема, токены из `:root`).
- **Деплой:** Render (web-сервис + worker). Секреты в env Render. Локально — `.env` (в `.gitignore`).

## 6. Пайплайн обработки (стадии)

1. **normalize** — из текста/файла регуляркой вытащить все `instagram.com/(reel|p|tv)/<shortcode>`,
   привести к канону, определить `type`. Для `message_1.json` — распарсить вложенные `share`-объекты
   (учесть mojibake latin1→utf8).
2. **dedup + cache** — отбросить дубли внутри партии по shortcode; для уже существующих shortcode с
   `transcripts.status='done'` — НЕ обрабатывать, переиспользовать.
3. **enqueue** — создать `reels` (если новые) и `jobs` со `state='queued'`; вернуть `session_id`.
4. **download** (воркер) — yt-dlp, **троттлинг 1–2 параллельно + паузы с джиттером**. Ошибки → backoff-ретрай;
   после N попыток → Apify-фолбэк или `failed` с причиной.
5. **extract_audio** — ffmpeg → mono 16 kHz; видео НЕ сохранять.
6. **transcribe** — faster-whisper выбранной моделью; авто-определение языка.
7. **metadata** — просмотры/лайки/комменты/автор/подписчики (из yt-dlp/Apify), если `pull_stats`.
8. **translate** — если язык ≠ ru и `translate=true` → `text_ru`.
9. **summarize** (опц.) — LLM делает `summary` + `tags`.
10. **ocr** (опц., фаза 3) — текст, вшитый в кадры.
11. **persist** — записать `transcripts`, обновить `reels`.
12. **cleanup** — удалить временные аудиофайлы.

Каждая стадия идемпотентна и логирует прогресс; статусы видны фронту.

## 7. API (FastAPI)

| метод | путь | назначение |
|---|---|---|
| POST | `/api/import` | принять ссылки/текст/содержимое файла + настройки (engine, model, translate, pull_stats, comment); распарсить, дедуп, создать сессию и задания; вернуть `session_id`, counts (reels/posts/dup) |
| GET | `/api/sessions` | список сессий импорта (история загрузок) |
| GET | `/api/sessions/{id}` | сессия + прогресс (`loaded/total`, `failed`) |
| PATCH | `/api/sessions/{id}` | изменить `comment` сессии |
| GET | `/api/reels` | список с метриками; query: `session`, `q`, `filter`(all/viral/done/failed), `author`, `sort`, `dir`, пороги `min_views/min_likes/min_comments/min_followers/min_er`, `page` |
| GET | `/api/reels/{id}` | детали: caption, transcript, text_ru, метрики, summary, note |
| PATCH | `/api/reels/{id}/note` | сохранить личную заметку |
| POST | `/api/reels/{id}/summary` | сгенерировать саммари (LLM) |
| GET | `/api/export` | query: `format`(csv/xlsx/json/md), `lang`(orig/ru/both), `scope`(all/selected), `ids`; отдаёт файл |
| GET | `/api/progress?session=` | SSE/poll: `loaded`, `total`, `failed`, новые готовые id |

`/api/import` идемпотентен по shortcode. Прогресс должен поддерживать «ранний показ»: как только
готова первая партия (~50), фронт уже может запрашивать `/api/reels`, а `/api/progress` досчитывает остальное.

## 8. Движки распознавания

- **faster-whisper (дефолт).** Модели:
  - `medium` — быстро, идёт на MacBook M1, качество хорошее. Дефолт для повседневного добивания новых рилсов.
  - `large-v3` — максимальная точность на русском/сложном звуке; тяжелее. Рекомендуемый сценарий —
    разовый батч на **бесплатном GPU (Kaggle ~30 ч/нед, влезает в 16 ГБ; либо Colab)**, результат импортируется обратно.
- **Yandex SpeechKit** — облачный фолбэк, доступен из РФ.
- **Deepgram / OpenAI Whisper** — опциональные облачные движки (выбор в UI).

Выбор движка и модели — на экране импорта (см. прототип: крупные карточки medium/large-v3, у large-v3
пометка про бесплатный Kaggle GPU). Облачные движки сами задают размер модели — карточки medium/large скрывать.

## 9. Метрики и формулы

| метрика | формула | смысл | отображение |
|---|---|---|---|
| 🔥 Залётность `er` | `views / followers` | во сколько раз охват превысил аудиторию автора | `N×`; оранжевый ≥10, зелёный 5–10, серый <5 |
| ❤/👥 Лайки/подп `lpf` | `likes / followers · 100%` | активность аудитории автора | `%` |
| 💬/👥 Комм/подп `cpf` | `comments / followers · 100%` | провоцирует ли обсуждение | `%` |
| 💞 Вовлечённость `eng` | `(likes+comments) / views · 100%` | доля посмотревших, кто отреагировал | `%` |

Фильтр «🔥 Залётные» = `er ≥ 5`. Формат чисел: `420 тыс`, `1,2 млн`.

## 10. Фронтенд — экраны и поведение (повторять `reelscribe_v12.html`)

Степпер «Импорт → Обработка → Результаты» в шапке; **на экране результатов степпер скрыт**.

### 10.1 Импорт
- Зона вставки: textarea для ссылок + drag&drop файла (`.txt`, `.csv`, сырой `message_1.json`).
  Парсинг регуляркой на лету; панель «найдено» (рилсы / посты / дублей убрано) + предпросмотр.
- Блок «🎙 Чем распознавать»: селект движка + **две крупные карточки medium / large-v3** (показываются
  только для faster-whisper), у large-v3 — пометка про бесплатный Kaggle GPU.
- Селект языка; чекбоксы «Переводить иностранные на русский», «Подтягивать просмотры/автора».
- **🗒 Комментарий к этой загрузке** (зачем гружу) — пишется в `import_sessions.comment`.
- **🕓 История загрузок** — список прошлых сессий (дата, кол-во, движок, статус, комментарий), кнопка «Открыть».
- Кнопки «Начать транскрибацию» и «Демо».

### 10.2 Обработка
- Грузит первую партию (~50) и быстро открывает результаты. Прогресс-бар, плитки статусов
  (скачивание/распознавание/готово/ошибки), живая лента. Подпись про троттлинг.

### 10.3 Результаты
Три режима через переключатель **Таблица · 📊 Аналитика · ▤ Чтение**, активный — залит акцентом.
**По умолчанию открывается «Аналитика».**

- **Одна таблица** для «Таблица» и «Аналитика»: режим аналитики **добавляет** столбцы (автор,
  подписчики, лайки, комменты, ❤/👥, 💬/👥, 💞), не заменяя текстовые; широкая таблица скроллится горизонтально.
- **Столбцы:**
  - порядок по умолчанию: `тип, дата, текст поста, расшифровка, автор, просмотры, 🔥, подписчики, лайки, комменты, ❤/👥, 💬/👥, 💞, статус`;
  - **drag-and-drop** заголовков — менять местами;
  - сортировка кликом по заголовку;
  - настройка видимости — кнопка **🧱 Столбцы** (чекбоксы);
  - числовые/процентные столбцы — **узкие, заголовок только иконка** (👁 🔥 ❤️ 💬 👥 ❤/👥 💬/👥 💞),
    при наведении — тултип с расшифровкой и формулой;
  - **цветовые группы:** охват (👁, 🔥) — янтарный; сырые (👥, ❤️, 💬) — бирюзовый; соотношения (❤/👥, 💬/👥, 💞) — фиолетовый;
  - на ячейках «текст поста» и «расшифровка» — тултип с полным текстом при наведении; если есть заметка — значок 📝.
- **Поиск** по подписи и расшифровке; быстрые чипы: Все / 🔥 Залётные / Готово / Ошибки.
- **⚙ Фильтры:** автор (выпадающий список), сортировка (дата / автор А-Я / просмотры / подписчики /
  залётность / лайки / комменты / ❤/👥 / 💬/👥 / вовлечённость), пороги-минимумы, «Сбросить».
- **🌐 Перевод на русский** — тумблер: в таблице иностранные расшифровки показываются по-русски (метка RU).
- **Карточка рилса (выезжает справа по клику на строку ИЛИ по нику автора).** Порядок:
  1. **превью видео** (вертикальное 9:16, нейтральный тёмный фон, play → ведёт на рилс в Instagram);
  2. автор (аватар-инициал, @ник, число подписчиков, дата);
  3. **📄 Текст поста** — карточка с акцентной полосой, **сворачивается** (по умолчанию развёрнута);
  4. **📝 Расшифровка** — светлая «бумага»; для иностранных по умолчанию **русский перевод** +
     кнопка «Показать оригинал»;
  5. **плитки метрик** — иконка крупно по центру сверху, под ней значение (порядок: просмотры,
     подписчики, лайки, комменты, 🔥, ❤/👥, 💬/👥, 💞); тултип на каждую с формулой;
  6. саммари (кнопка «Саммари через Claude») + теги;
  7. **🗒 Моё примечание** (к рилсу) + «Сохранить».
- **Режим Чтение:** слева список, справа читалка — превью-обложка, автор, метрики, расшифровка крупным
  шрифтом, переключатель оригинала, поле заметки, навигация **‹ Предыдущий / Следующий ›** со счётчиком «N из M».
- **Экспорт (модал):** форматы CSV / Excel / JSON / Markdown / Supabase (MCP) / Notion; живое превью;
  **заголовки колонок на русском** (в JSON — техн. ключи); выбор **языка текста: оригинал / на русском / оба**
  (по умолчанию «на русском»); объём — все готовые / только выбранные.

> Запрет: в артефактах/фронте не использовать `localStorage`/`sessionStorage` для прод-хранения — состояние
> в React-стейте на сессию, постоянное хранение — в Supabase (см. §12).

## 11. Перевод

- Язык оригинала определяется ASR. Если ≠ ru и включён перевод — генерируем `text_ru`.
- В UI иностранное по умолчанию показывается на русском; оригинал — по кнопке. Оба сохраняются.
- В экспорте — выбор языка (оригинал/русский/оба), дефолт русский.

## 12. Кэш, история, заметки (персистентность)

- **Кэш по shortcode:** повторный импорт того же рилса не пересчитывается — берётся готовая расшифровка.
- **История загрузок:** каждая сессия в `import_sessions` со своим комментарием; видна на экране импорта,
  можно «Открыть» (показать её результаты).
- **Заметки двух уровней:**
  - по рилсу — `reel_notes.note` (значок 📝 в таблице, колонка «моя заметка» в экспорте);
  - по сессии — `import_sessions.comment` (полоса под шапкой результатов).

## 13. Нефункциональные требования

- **Троттлинг и устойчивость:** 1–2 параллельных скачивания, паузы с джиттером, backoff-ретраи,
  пер-рилсовая изоляция ошибок, фолбэк Apify.
- **Идемпотентность:** импорт по shortcode; повторный запуск не плодит дубли и не пересчитывает готовое.
- **Безопасность:** секреты только в env; Supabase RLS; не логировать ключи.
- **Производительность:** ранний показ первой партии; пагинация `/api/reels`; индекс по `shortcode`,
  `author_handle`, метрикам сортировки.
- **Память:** на M1 крутить `medium`; `large-v3` — батчем на внешнем GPU.
- **Логирование:** прогресс по стадиям, причины `failed`.

## 14. Дорожная карта

### MVP (Фаза 1) — «ссылки → расшифровки → таблица → экспорт»
- Импорт (вставка/`.txt`/`.csv`/`message_1.json`) → normalize → dedup/cache → очередь.
- Скачивание (троттл) → аудио → **faster-whisper `medium`** → persist → cleanup (видео не хранить).
- Результаты: таблица + поиск + карточка рилса (превью, текст поста, расшифровка) + детали.
- Экспорт CSV/JSON с **русскими заголовками**. UI на русском. Кэш по shortcode. История сессий (базовая).
- Ранний показ первой партии + фоновая догрузка.

### Фаза 2 — аналитика и удобство

## 15. Критерии приёмки MVP

1. Дропнул `message_1.json` → корректно распознаны ссылки, убраны дубли, создана сессия.
2. Партия обрабатывается с троттлингом, первая часть результатов видна рано, остальное догружается.
3. У рилса есть `caption`, `transcript`, метрики; клик открывает карточку с превью и текстами.
4. Повторный импорт того же рилса не пересчитывается (кэш по shortcode).
5. Экспорт CSV даёт **русские заголовки**, текст по умолчанию на русском, видео нигде не сохранено.
6. Весь интерфейс на русском; ошибки отдельных рилсов не валят всю партию.

## 17. Инструменты разработки (Claude Code)

Проект ведётся в Claude Code с применением современных возможностей. Подробное объяснение каждого
инструмента (что это, как включить, как работает) — в `docs/claude-code-toolkit.md`. Операционные
правила «как строить» — в `CLAUDE.md`. Ниже — привязка инструментов к этапам этого ТЗ.

| Этап (раздел ТЗ) | Инструмент Claude Code | Зачем здесь |
|---|---|---|
| Скелет, модель данных (§5), контур API (§7) | plan mode + `/effort high` + Supabase MCP | план и схема БД до кода; таблицы — через MCP |
| Пайплайн (§6): normalize→download→audio→transcribe→persist | субагент «backend» | крупная изолированная часть отдельным агентом |
| Фронтенд (§10): таблица, карточка, режимы | субагент «frontend» + скилл-сверка с `reelscribe_v12.html` | параллельно с бэкендом; UI строго по прототипу |
| Движки ASR (§8) и downloader-стратегии | `/effort xhigh` | проектирование абстракций — самая «думательная» часть |
| Тесты пайплайна и сервисов | хуки (`.claude/settings.json`) | авто-линтер/тесты без ручного запуска |
| Работа с Postgres: миграции, проверочные запросы | Supabase MCP | прямой доступ к базе из сессии (с подтверждением DDL) |
| Добивка MVP до критериев приёмки (§15) | `/goal` | автономная работа до измеримой цели |
| Ревью перед коммитом и деплоем на Render | `/code-review` (на сложном — `high`) | поймать баги до прода |
| Крупный рефактор (напр. вынос правил в `.claude/rules/`, §16) | динамические воркфлоу `/workflows` | разбить большую задачу на много шагов |
| Секреты, опасные команды, миграции | auto mode + подтверждения | безопасное само-одобрение, стоп на необратимом |

**Ограничение по железу.** Разработка идёт на MacBook M1 (8 ГБ): параллелизм субагентов — максимум 2–3;
не запускать faster-whisper/ffmpeg/сборку фронта в нескольких агентах одновременно.

**Режим обучения.** Перед первым применением каждого инструмента Claude Code объясняет его (что/зачем
здесь/как включить/как работает) и вводит инструменты по 1–2 за этап — см. `CLAUDE.md`.
